# 📝 브라우저의 자료 저장소

<br></br>
## ✅ 자료 저장소?
브라우저에는 여러 유형의 자료들을 저장하는 저장소가 있으며, 아래와 같이 분류할 수 있다.
```
브라우저 저장 공간
 ├── 캐시
 └── 브라우저 저장소
      ├── 쿠키
      ├── 웹 스토리지
      │    ├── 로컬스토리지
      │    └── 세션스토리지
      └── IndexedDB
```
<br></br>
### 📖 캐시(Cache)
- 시스템을 향상시키기 위해, 웹 서버 요청에 응답받은 리소스 파일들을 빠르게 불러오기 위한 임시 저장소.
- HTML 초기버전부터 사용하던 저장소
- 브라우저 캐시, 서비스워커 캐시, HTTP 캐시 등 여러 종류의 캐시가 있음

#### ❓ 활용 예시
- 이전에 접속했던 웹사이트 접속
  
  > 서버에 다시 요청을 보내지 않고, 이전에 캐시에 저장해둔 리소스를 불러와 웹페이지를 불러온다.

<br></br>
### 📖 쿠키(Cookie)
- 클라이언트 측에 저장되는 텍스트파일
- HTML5 이전부터 존재하던 저장소
- 클라이언트-서버 간 데이터를 저장하며, HTTP 헤더와 함께 서버로 전송됨
- 저장 가능한 용량이 작고, 만료 시간을 지정 가능

#### ❓ 활용 예시
- 로그인 유지 기능
  
  > 사용자가 로그인을 할 때, 서버에서 세션ID를 발급하며 이것을 쿠키에 저장한다.
  >
  > 이후 요청마다 서버에 세션ID를 전송하여 로그인 상태임을 알린다.

<br></br>
### 📖 로컬 스토리지(Local Storage)
- 클라이언트 측에 데이터를 저장하는 간단한 저장소
- HTML5에 신규 도입
- 탭/창 관계없이 같은 브라우저라면 접근 가능
- 브라우저를 종료해도 데이터가 유지됨
- 5~10MB 정도의 데이터를 저장 가능

#### ❓ 활용 예시
- 로컬 스토리지는 중요하지 않은 개인 정보 저장에 적합함

|활용 분야|예시|
|---|---|
|사용자 개인 설정 저장|언어 설정, 글꼴 크기 등|
|임시 데이터 저장|폼 작성 중 데이터를 임시 저장(글 작성 중 브라우저 새로고침 대비)|
|사용자 프로필|사용자 이름, 프로필 이미지 등 민감하지 않은 정보만 저장|
|최근 본 항목|최근 검색어, 방문 기록 등|


<br></br>
### 📖 세션 스토리지(Session Storage)

- 클라이언트 측에 데이터를 저장하는 간단한 저장소
- HTML5에 신규 도입
- 같은 브라우저라도 탭/창 별로 데이터가 관리됨
- 브라우저 탭/창을 닫으면 데이터가 사라짐
- 임시 데이터를 저장할 때 활용됨

#### ❓ 활용 예시
- 세션 스토리지는 탭이 활성화 되어있을 동안만 필요한 임시 데이터 저장에 적합함

|활용 분야|예시|
|---|---|
|임시 로그인|탭이 열려있을 동안만 로그인이 유지되도록 임시 로그인 상태 저장|
|페이지 이동 간 데이터 전달|페이지를 이동할 때 간단한 데이터(예: 선택한 데이터)를 저장해서 가져오기|

<br></br>
### 📖 IndexedDB
- 클라이언트 측에 구조화된 대용량 데이터를 저장하는 데이터베이스
- HTML5에 신규 도입
- 비동기 API 기반이며 수백 MB ~ GB 단위 데이터를 저장가능
- 복잡한 쿼리를 이용한 데이터검색 가능

#### ❓ 활용 예시
- IndexedDB는 오프라인 작업상태나 크고복잡한 데이터를 구조적으로 저장하는데 적합함

|활용 분야|예시|
|---|---|
|오프라인 노트|작성한 노트 데이터를 저장하고, 온라인 상태가 되면 서버로 싱크|
|HTML5 게임|게임 세이브 파일을 저장하고, 플레이어가 중간 저장/불러오기|

<br></br>

## ✅ 요약

|저장소 종류|용도|
|---|---|
|캐시(Cache)|웹 서버에서 받은 리소스를 저장|
|쿠키(Cookie)|서버와 주고받는 데이터를 저장|
|로컬스토리지(Local Storage)|브라우저 안에 오래 남는 데이터 저장|
|세션스토리지(Session Storage)|탭/창에만 필요한 임시 데이터 저장|
|IndexedDB|대용량, 복잡한 데이터를 브라우저에 저장|
